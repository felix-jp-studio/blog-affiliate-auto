## 1. 「訪問性」の定義と目標

本ロードマップでいう **訪問性** は、次の 4 段階の漏斗で測る。

```
クロール可能 → インデックス登録 → 検索表示（imp） → クリック（visit）
```

| 段階           | 現状（2026-07-22）   | Week 4 目標       | Month 2 目標      | Month 4 目標  |
| -------------- | -------------------- | ----------------- | ----------------- | ------------- |
| 公開 URL 数    | 36 記事 + 7 Hub      | 50                | 80                | 150           |
| インデックス数 | ≥1（トップ確認済）   | **15+**           | **40+**           | **90+**       |
| 月間表示回数   | 1（28 日）           | 100–500           | 3,000–8,000       | 15,000+       |
| 月間クリック   | 0                    | 5–30              | 80–200            | 400+          |
| 平均 CTR       | —                    | 1–3%              | 2–4%              | 3–5%          |
| 平均掲載順位   | 19（参考・データ少） | 25 位以内 KW 5 件 | 11–30 位 KW 20 件 | TOP10 KW 3 件 |

**収益ロードマップとの関係**: 訪問性 KPI が先、収益 KPI（ASP 成約）は Month 4 以降が本格化。Week 3–4 は **100% 訪問性優先**。

## 3. 核心課題（訪問性ボトルネック）

1. **インデックス遅延**: 公開 2 週目・36 本中ほとんど未登録。表示が出る前に順位改善は不可能。
2. **供給はあるが「見つかる」仕組みが弱い**: 週 7 本生成は回っているが、robots.txt・IndexNow・公開後 ping がない。
3. **CTR 最適化未着手**: タイトル / description / 構造化データが Article のみ。FAQ 記事が多いのに FAQPage なし。
4. **リライトループ未稼働**: GSC CSV が必要。11–30 位 KW の伸ばし方が未定義。
5. **カテゴリ偏り**: sim/hikari は厚いが trouble/cost が薄く、ロングテールの幅が狭い。

## 6. 週次オペレーション（訪問性 PDCA）

**毎週日曜 20 分**（ASP レポート 15 分と合わせて 35 分）

| #   | チェック                                  | 記録先                             |
| --- | ----------------------------------------- | ---------------------------------- |
| 1   | GSC → インデックスページ数（Pages）       | `gsc-baseline.md` 追記 or 週次ログ |
| 2   | 表示 / クリック / CTR / 平均順位（28 日） | 同上                               |
| 3   | 新規公開記事の URL 検査（今週分）         | チェックリスト                     |
| 4   | クエリ TOP10 変化                         | リライト候補メモ                   |
| 5   | GA4 → オーガニックセッション              | GA4 標準レポート                   |

**判断基準**

| 兆候                          | アクション                                          |
| ----------------------------- | --------------------------------------------------- |
| Month 2 末でインデックス < 20 | 週 7 → 5 に減速、文字数・独自要素追加、URL 検査強化 |
| 表示はあるが CTR < 1%         | meta / 構造化データ / タイトルリライト優先          |
| 特定記事だけ imp ゼロ         | canonical・noindex 誤り・重複を E2E で確認          |
| thin content 警告             | 生成プロンプトに独自比較表・FAQ 必須化              |
