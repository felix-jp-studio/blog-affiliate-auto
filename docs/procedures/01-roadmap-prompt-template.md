# 手順 01：ロードマップ生成プロンプト（テンプレート）

## 目的

ブログアフィリエイトの全自動化と 12 ヶ月での収益化に向けた**詳細ロードマップ**を AI に生成させる。

## 前提

- 以下のプロンプトをそのまま AI チャットにコピペする
- 【】内の変数を自分の条件に合わせて編集する
- 生成結果は別ファイル（例: `docs/plans/roadmap-YYYYMMDD.md`）に保存することを推奨

## 実行手順

1. 下記「プロンプト本文」の【】をすべて埋める
2. （任意）手順書 README の一行追記を先頭に追加
3. AI に貼り付けて送信
4. 出力をレビューし、月次 KPI・自動化構成に抜けがないか確認
5. 確定版を `docs/plans/` に保存（今後追加）

---

## プロンプト本文

```markdown
# 役割

あなたはブログアフィリエイトとSEO、コンテンツ自動化、収益化戦略に精通したシニアコンサルタントです。
「人手を最小限にしつつ、1年以内に安定したアフィリエイト収益を得る」ことを前提に、
現実的かつ実行可能なロードマップを設計してください。

# 私の前提条件（※必要に応じて編集）

- 目標期間：12ヶ月
- 目標月間収益：【例：5万円 / 10万円 / 30万円】
- 初期投資上限：【例：月1万円 / 5万円 / 10万円】
- 1日あたりの作業時間：【例：30分 / 1時間 / ほぼゼロ（完全自動化志向）】
- 得意分野・関心ジャンル：【例：ガジェット / 美容 / 転職 / 子育て / 未決定】
- 技術スキル：【例：プログラミング可 / WordPressのみ / ノーコードのみ】
- 使用予定ツール（任意）：【例：WordPress, ChatGPT, Claude, Make/Zapier, Ahrefs, GA4】
- 対象市場：【例：日本国内 / 英語圏 / 両方】
- リスク許容度：【例：低（堅実） / 中 / 高（攻め）】

# 依頼内容

上記前提を踏まえ、以下の構成で**12ヶ月の詳細ロードマップ**を作成してください。

## 1. 戦略サマリー（1ページ相当）

- 推奨ニッチとその選定理由（競合・単価・検索ボリューム・自動化しやすさ）
- 収益モデル（ASP、Amazon、楽天、個別提携など）
- 自動化の設計思想（何を自動化し、何は人が残すか）
- 12ヶ月後の到達イメージ（記事数、PV、収益、運用工数）

## 2. フェーズ別ロードマップ（月次）

各月について以下を必ず記載：

- **主要ゴール**
- **やること（タスク一覧）**
- **使用ツール・自動化フロー**（可能ならフロー図 or 箇条書き）
- **KPI**（記事数、インデックス数、PV、CTR、CVR、収益、作業時間）
- **判断基準**（続行 / 改善 / 撤退の分岐点）
- **想定リスクと対策**

フェーズは最低でも以下に分割：

- Phase 0（準備期：0〜1ヶ月）
- Phase 1（基盤構築：1〜3ヶ月）
- Phase 2（量産・検証：4〜6ヶ月）
- Phase 3（収益化加速：7〜9ヶ月）
- Phase 4（最適化・スケール：10〜12ヶ月）

## 3. 自動化アーキテクチャ設計

以下を具体的に設計：

1. **キーワード調査の自動化**
2. **記事企画・構成の自動化**
3. **記事生成（AI）と品質管理**
4. **画像・メタ情報・内部リンクの自動化**
5. **WordPress/CMSへの自動投稿**
6. **アフィリエイトリンク挿入ルール**
7. **インデックス促進・更新サイクル**
8. **アクセス解析と改善ループ**
9. **収益レポート自動化**

各項目について：

- 推奨ツール（無料/有料）
- 実装難易度（低/中/高）
- 月額コスト目安
- 失敗しやすいポイント

## 4. コンテンツ戦略

- 記事タイプ設計（比較記事、レビュー、How-to、ランキング等）
- 1記事あたりの品質基準（文字数、E-E-A-T、独自性）
- 月間投稿本数目標（フェーズごと）
- 収益化に直結する記事比率（例：比較記事70%）
- ニッチ内での勝ち筋キーワード例（30個）

## 5. SEO・集客戦略

- 初期ドメイン戦略（新規/既存、サブドメイン可否）
- 内部リンク設計
- 被リンク方針（自動化可能範囲と手動範囲）
- SNS/メルマガ/ショート動画の役割（やる/やらない）
- Googleアルゴリズム更新への耐性設計

## 6. 収益化設計

- 推奨ASPと提携優先順位
- 記事カテゴリ別の単価・CVR想定
- 収益シミュレーション（保守/標準/楽観の3パターン）
- 月次損益分岐点（何記事・何PVで黒字化か）

## 7. 運用ルール（人間が残す最小タスク）

- 週次でやること（30分以内）
- 月次でやること（2時間以内）
- 品質監査チェックリスト（10項目）
- 自動化してはいけない領域（コンプラ・品質・法務）

## 8. 12週間の実行プラン（クイックスタート）

最初の90日を「週単位」で具体化：

- 毎週のToDo
- 完了条件
- 使うテンプレート（記事構成、レビュー観点、投稿チェック）

## 9. 失敗パターンと回避策

- よくある失敗10選（薄いAI記事、ASP規約違反、ニッチミス、過剰自動化など）
- それぞれの早期警告サイン
- リカバリー手順

## 10. 最終成果物フォーマット

最後に以下を表形式で出力：

- 月次ロードマップ一覧表（月 / 目標 / KPI / 主要タスク / 予算 / 工数）
- 自動化パイプライン一覧表（工程 / ツール / コスト / 難易度 / 優先度）
- 収益予測表（月次、3シナリオ）

# 制約・出力条件

- 現実的な数字を使う（根拠も簡潔に記載）
- 「完全自動化」を謳いつつ、品質と規約遵守のために人間確認が必要な箇所は明確化
- 初心者でも実行できる順序で記述
- 抽象論だけで終わらず、**具体タスク・ツール名・テンプレート例**まで落とし込む
- 日本市場前提（必要ならグローバル展開案も添える）
- 最後に「最初の7日間でやるべき10個のタスク」を優先度順で提示

# 追加で提案してほしいこと

- 私の前提条件に対する最適ニッチ候補を3案
- 低予算版と高予算版の2パターン比較
- 3ヶ月時点の中間レビュー質問リスト（20問）
- 次に深掘りすべき論点（私が追加質問すべき項目）

では、上記に基づきロードマップを作成してください。
```
