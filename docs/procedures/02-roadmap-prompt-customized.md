# 手順 02：ロードマップ生成プロンプト（確定条件版）

## 目的

[確定済み前提条件](#確定済み前提条件) を埋め込んだロードマップ生成プロンプトを AI に投入し、12 ヶ月の実行計画を得る。

## 確定済み前提条件

| 項目         | 値                                                                                                                                            |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------- |
| 目標期間     | 12 ヶ月                                                                                                                                       |
| 目標月収     | 最低 5 万円 / ベスト 10 万円                                                                                                                  |
| 初期投資     | 月 1〜3 万円想定（要提案）                                                                                                                    |
| 作業時間     | 平日 1 時間/日、休日は要調整（週 5〜9 時間）                                                                                                  |
| 技術         | TypeScript / Python、API・CI 連携可                                                                                                           |
| ジャンル     | 未決定 → [niche-pre-research.md](../research/niche-pre-research.md) 推奨 **案1（格安SIM×光回線）** を採用する場合はプロンプト内ニッチ欄を更新 |
| 記事作成     | 全自動（キーワード → 生成 → 投稿。リスク承知）                                                                                                |
| 対象市場     | 日本国内                                                                                                                                      |
| リスク許容度 | 中〜高（アカウント停止・規約違反は回避）                                                                                                      |

## 実行手順

1. ニッチを確定している場合、プロンプト内の「ジャンル」行を書き換える
2. 下記プロンプト全文を AI にコピペ
3. （任意）先頭に一行追記：`【重要】回答は日本語。実行可能な具体性を最優先。各月のKPIは数値必須。自動化は Make/Zapier/API連携まで具体化。`
4. 出力を `docs/plans/roadmap-YYYYMMDD.md` に保存（今後作成）

## プロンプト本文

```markdown
# 役割

あなたはブログアフィリエイト、SEO、AIコンテンツ自動化、収益化戦略に精通したシニアコンサルタント兼テクニカルアーキテクトです。
「人手を最小限にしつつ、12ヶ月以内に安定したアフィリエイト収益を得る」ことを前提に、
現実的かつ実行可能なロードマップを設計してください。

# 私の前提条件（確定事項）

- 目標期間：12ヶ月
- 目標月間収益：
  - 最低ライン：5万円/月（12ヶ月目までに達成必須）
  - ベストライン：10万円/月（12ヶ月目の理想目標）
- 初期投資上限：月1〜3万円を想定（過剰投資は避けたい）
- 1日あたりの作業時間：
  - 平日：1時間/日
  - 休日：要調整（繁忙期は少なめ、空いている週末は追加投入可）
  - 週あたりの実働目安：平日5時間＋休日0〜4時間（合計5〜9時間/週）
- 技術スキル：
  - TypeScript、Python が書ける
  - 補助があれば他言語も対応可能
  - API連携、スクリプト自動化、CI/CD、CMS連携の実装が可能
- ジャンル：未決定（あなたが3案提案し、私が選定する前提で設計）※確定済みの場合はここを書き換え
- 対象市場：日本国内（SEOはGoogle日本向け）
- 記事作成方針：
  - **完全自動化を最優先**（キーワード調査 → 構成 → 本文生成 → 画像/メタ → 投稿まで無人化）
  - AI生成記事の品質リスク・SEOリスク・ASP規約リスクは承知の上
  - 人間が残す作業は「監視・例外対応・月次改善」に限定したい
- リスク許容度：中〜高（自動化優先。ただしアカウント停止・規約違反は回避）

# 依頼内容

上記前提を踏まえ、以下の構成で**12ヶ月の詳細ロードマップ**を作成してください。
私はエンジニアなので、**ノーコードだけでなく TypeScript/Python による自作パイプライン**も積極的に提案してください。

## 1. 戦略サマリー（1ページ相当）

- 推奨ニッチ **3案**（競合・単価・検索ボリューム・自動化しやすさ・規約リスクで比較表）
- 各ニッチの「5万円/月」「10万円/月」到達に必要な概算（記事数・月間PV・CVR）
- 収益モデル（ASP、Amazon、楽天、個別提携の優先順位）
- 自動化の設計思想（何を100%自動化し、何を週1回の人間チェックに残すか）
- 12ヶ月後の到達イメージ（記事数、月間PV、収益、週あたり作業時間）

## 2. フェーズ別ロードマップ（月次・12ヶ月）

各月について以下を必ず記載：

| 項目         | 内容                                                 |
| ------------ | ---------------------------------------------------- |
| 主要ゴール   | 数値目標付き                                         |
| タスク一覧   | チェックボックス形式                                 |
| 自動化フロー | 工程とデータの流れ                                   |
| 使用ツール   | 無料/有料、代替案                                    |
| KPI          | 記事数、インデックス数、PV、CTR、CVR、収益、作業時間 |
| 判断基準     | 続行 / 改善 / ピボット / 撤退の分岐点                |
| リスクと対策 | 具体的に                                             |

フェーズ分割：

- **Phase 0**（準備：0〜1ヶ月）— ニッチ選定、ドメイン、基盤構築
- **Phase 1**（基盤：1〜3ヶ月）— 自動化パイプライン v1、初期記事50〜100本
- **Phase 2**（量産・検証：4〜6ヶ月）— 完全無人投稿、SEO検証、収益化開始
- **Phase 3**（加速：7〜9ヶ月）— 勝ちキーワード拡大、月5万円ライン狙い
- **Phase 4**（最適化：10〜12ヶ月）— 月10万円ライン、運用コスト最小化
```
