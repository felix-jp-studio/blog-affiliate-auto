# 手順 03：ニッチ選定リサーチ

## 目的

ブログアフィリエイト全自動化プロジェクト向けに、**収益化しやすく AI 自動化と相性の良いニッチ**を 3 案に絞り、比較表と推奨 1 位を決める。

## 前提条件（本プロジェクト）

| 項目 | 値 |
|------|-----|
| 目標月収 | 最低 5 万 / ベスト 10 万（12 ヶ月目） |
| 作業時間 | 平日 1h/日 |
| 技術 | TS / Python 可 |
| 記事 | 全自動（リスク承知） |
| 市場 | 日本国内 |

## 実施手順

### Step 1：評価軸を固定する

以下 6 軸で各候補を 1〜5 点で採点する。

| 軸 | 見るポイント |
|----|-------------|
| 自動化適性 | 公開情報・比較表中心か、体験談必須か |
| 成果単価 | ASP 案件の平均単価（手数料前） |
| 5 万到達難易度 | 必要 PV・成約数が 12 ヶ月で現実的か |
| 競合 | ヘッド KW の強さ / ロングテールの豊富さ |
| 規約リスク | YMYL・薬機法・景表法・ASP 却下 |
| 技術活用 | スクレイピング・API 連携が効くか |

**除外ルール（AI 全自動前提）**

- YMYL 本丸（医療・投資・保険の専門判断）は原則除外
- 体験談必須（脱毛・ジム等）は優先度を下げる

### Step 2：候補ジャンルを 5〜7 個リストアップ

例：

- 格安 SIM / 光回線
- プログラミングスクール / IT リスキル
- ビジネス SaaS / 副業ツール
- VOD / サブスク比較
- ガジェット・家電
- 転職（一般）
- クレジットカード

### Step 3：市場データを収集する

各候補について以下を調べる。

1. **ASP 単価** — A8.net、もしもアフィリエイト、Link-AG 等の案件表
2. **実売サイト** — ラッコ M&A、公開ブログ収益レポート
3. **AI 相性** — 事実情報比率、更新頻度、競合の記事タイプ
4. **検索 KW** — ラッコキーワード等でロングテール候補 10 個

### Step 4：3 案に絞り込む

Step 2 のうち、総合点が高い 3 案を選ぶ。  
各案に以下を記載：

- 概要（1 段落）
- 成果単価表
- 月 5 万・10 万の逆算（成約数・PV）
- 自動化適性（1〜10）
- リスク表
- キーワード例（15〜30 個）
- 12 ヶ月の記事数目安

### Step 5：推奨 1 位を決める

判断基準の優先順位（本プロジェクト）：

1. 12 ヶ月で月 5 万の再現性
2. AI 全自動との相性
3. エンジニアスキルの活用度
4. 規約リスクの低さ

### Step 6：成果物を保存する

```
docs/research/niche-pre-research-YYYYMMDD.md
```

テンプレートは [niche-pre-research.md](../research/niche-pre-research.md) をコピーして更新する。

### Step 7：次の手順へ

- 推奨ニッチを [02-roadmap-prompt-customized.md](./02-roadmap-prompt-customized.md) の「ジャンル」欄に反映
- ロードマップ生成プロンプトを実行
